## Faz 3 — Codebase Findings

### Bug A — Çift Canonical/Hreflang (KÖK SEBEP)
- `app/[locale]/layout.tsx:152` → `<HreflangLinks locale={locale} path={hreflangPath} />` — her sayfada render ediliyor
- `components/seo/HreflangLinks.tsx:3-18` — bir canonical + 10 hreflang emit ediyor
- `app/[locale]/services/[slug]/page.tsx:76-79` — `generateMetadata` `alternates: { canonical, languages }` ile YİNE bir canonical + 10 hreflang emit ediyor (Next.js bunları otomatik render ediyor)
- Sonuç: her service detail page'inde **2 canonical, 2 set hreflang**.

### Bug B — Page-level metadata path-word'ü hardcoded
- `app/[locale]/services/[slug]/page.tsx:67-71` ve `:77`:
  ```ts
  languages[hreflangForLocale(l)] = `${SEO_BASE}/${l}/services/${slug}`;  // tüm 9 locale için /services/
  // ...
  canonical: `${SEO_BASE}/${locale}/services/${slug}`,
  ```
  Hâlbuki `lib/seo.ts` **doğru** bir `buildAlternates(locale, pathSuffix)` zaten exportluyor — next-intl `getPathname` üzerinden locale-bazlı path-word'ü çözüyor. Page bunu kullanmıyor.

### Bug C — Layout HreflangLinks request-path verbatim kullanıyor
- `app/[locale]/layout.tsx:26-32` `hreflangPathForRequest(locale)`:
  ```ts
  const pathname = headers().get("x-pathname") ?? "";
  // /de/dienstleistungen/math-tutoring → /dienstleistungen/math-tutoring
  return pathname.slice(prefix.length);
  ```
- `components/seo/HreflangLinks.tsx:12` bu path'i 9 locale prefix'iyle birleştiriyor: `${SEO_BASE}/${l}${path}`
- Her hedef locale CURRENT locale'ın path-word'üyle prefix'leniyor → cross-contamination

### Bug D — Cross-locale redirect'ler 307 (geçici), 308 (kalıcı) olmalı
- `next.config.mjs:48-223` `permanent: true` redirect'leri SADECE `/{locale}/services/...` (EN slug → localized slug) için tanımlı → 308
- `/{locale}/<diğer-locale-slug>/...` (örn. `/de/posluhy/...`, `/de/hizmetler/...`) next-intl middleware'a düşüyor, **307** dönüyor
- Google 307'leri "geçici" sayıp signal'ı destination'a aktarmıyor → "Page with redirect (206)" kovası

### Bug E — OG image dinamik route 404
- Next.js'de `app/[locale]/services/[slug]/opengraph-image.tsx` PNG generator route'u
- Google bu URL'i `<meta property="og:image">`'den alıp HTML olarak crawl etmeye çalışıyor
- Slug invalid olduğunda veya cache-buster query'sinde 404 dönüyor → 61 URL

### Bug F — Provider/Pros identity path
- `i18n/routing.ts:212-213`:
  ```ts
  "/pros/[slug]": "/pros/[slug]",
  "/provider/[id]": "/provider/[id]",
  ```
- 9 locale aynı URL pattern, içerik sadece çeviri farklı
- 9 locale × N pro = N×9 duplicate URL → "Crawled - not indexed (62)" + "Duplicate without canonical (402)" kovalarına dağılıyor

### Bug G — Sitemap temiz (eliminate)
- `app/sitemap.ts` `localizedWithParams` üzerinden next-intl `getPathname` çağırıyor → doğru locale-path-word üretiyor
- 2,394 URL canlı doğrulamada hepsi doğru locale-path eşleşmesi
- **Kaynak değil**

### Bug H — robots.txt sağlıklı
- `public/robots.txt` veya `app/robots.ts` (live curl): Disallow `/admin/`, `/dashboard/`, `/inbox/`, `/settings/`, `/api/`, `/auth/`, query string filtreleri, Sitemap referansı var
- Sorun değil
