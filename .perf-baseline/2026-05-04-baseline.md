## What was deliberately skipped

### Faz 5 — framer-motion LazyMotion swap
Audited: 63 files import `framer-motion`, 62 use `motion.*`, 21 use
`AnimatePresence`. A full LazyMotion + `motion.*` → `m.*` migration is
the right move for bundle size, but the surface area (62 files of
mostly-correct `motion` usage) plus the all-or-nothing nature of the
swap (any missed reference is a runtime error) is too high a blast
radius for a late-night sprint. Deferred to G-PERF-3 with explicit
scope:
- Add `<LazyMotion features={domAnimation}>` at the root layout
- Sweep all 62 files: `import { motion }` → `import { m as motion }`
  (the alias keeps the existing call sites unchanged at the JSX level)
- Verify each route still animates (visual regression sweep)

### Image priority changes
None warranted. The audit showed:
- `MobileMockup.tsx:99` already has `priority`
- `landing-page-client.tsx:295` (featured category cards) is below-fold
  with proper `sizes` attribute — adding `priority` would hurt LCP
- Hero LCP candidate is the `<h1>` text, not an image

### Hero refactor for `LinesGradient` / `AceternityHeroBackground`
`LinesGradient` is plain SVG, no framer-motion path. `AceternityHero-
Background` is inline `style` radial gradients — pure CSS, near-zero
JS cost. Splitting them would not move the LCP/TBT needle. Left as-is.
